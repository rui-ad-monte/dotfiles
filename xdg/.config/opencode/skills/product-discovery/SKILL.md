---
name: product-discovery
description: Gather implementation-ready product requirements from issues, proposals, and supporting docs
---

## What to produce

- Request summary
- Problem statement
- In scope
- Out of scope
- Acceptance criteria
- Dependencies
- Risks and unknowns
- Open questions
- Handoff notes for Architect

## How to work

- Prefer tracker and documentation facts over assumptions.
- Search for existing platform context before declaring missing information.
- Separate confirmed facts from inferred expectations.
- Keep the result implementation-ready, not solution-heavy.

## Quality bar

- Acceptance criteria must be testable.
- Scope boundaries must be explicit.
- Any ambiguity that changes implementation must be called out.
